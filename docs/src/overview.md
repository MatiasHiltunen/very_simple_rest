# VSR Book

VSR is a contract-first toolkit for building REST APIs from one source of truth.
The current center of gravity is the `.eon` service format plus the `vsr` CLI:
you define the service contract once, then use that same contract to serve the API,
generate migrations, emit a Rust server project, build a binary, publish OpenAPI,
generate a TypeScript client, and render reference docs.

This book exists to replace the old oversized root README with a navigable docs surface.
It intentionally keeps two documentation shapes:

- human-oriented chapters in `docs/src`
- a machine-scannable single-file reference generated by `vsr docs`

The `.eon` reference chapter in this book includes the checked-in generated file directly.
That keeps the local AI-facing markdown and the human-facing book on one source of truth.

## Local Docs Workflow

Regenerate the checked-in `.eon` reference:

```bash
cargo run -p vsra -- docs --output docs/eon-reference.md --force
```

Serve the book locally with `mdBook`:

```bash
mdbook serve docs
```

If you do not have `mdBook` installed yet:

```bash
cargo install mdbook --locked
```
